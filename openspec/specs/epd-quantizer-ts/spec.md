## ADDED Requirements

### 需求:6色调色板定义
量化引擎必须使用与 Waveshare 官方一致的 6 色调色板，索引对齐硬件颜色枚举。调色板数组索引 4（Orange）必须标记为跳过（`null`），不参与量化计算。

调色板定义：
- 索引 0 = BLACK (0, 0, 0)
- 索引 1 = WHITE (255, 255, 255)
- 索引 2 = YELLOW (255, 255, 0)
- 索引 3 = RED (255, 0, 0)
- 索引 4 = ORANGE（`null`，跳过，不参与量化）
- 索引 5 = BLUE (0, 0, 255)
- 索引 6 = GREEN (0, 255, 0)

#### 场景:调色板索引对齐
- **当** 量化引擎处理像素颜色
- **那么** 输出的颜色索引必须与 Waveshare 硬件 ColorSelection 枚举一致，索引 4 永远不会出现在量化结果中

#### 场景:调色板常量导出
- **当** 其他模块导入 palette.ts
- **那么** 必须能获取调色板 RGB 数组、颜色名称数组、ColorIndex 类型定义

### 需求:欧氏距离最近邻量化
量化引擎必须支持最近邻量化模式。对每个输入像素，必须计算其 RGB 值与调色板中每个有效颜色的欧氏距离平方（dr² + dg² + db²），选择距离最小的颜色索引。禁止使用 redmean 加权距离或其他变体。

#### 场景:纯色像素量化
- **当** 输入像素为 (255, 0, 0)（纯红）
- **那么** 量化结果必须为索引 3（RED）

#### 场景:中间色像素量化
- **当** 输入像素为 (128, 128, 128)（中灰）
- **那么** 量化引擎必须计算与所有 6 色的欧氏距离，返回距离最近的颜色索引（WHITE，因为距离最近 = 3 × 127² = 48387，小于 BLACK 的 3 × 128² = 49152）

### 需求:Floyd-Steinberg抖动量化
量化引擎必须支持 Floyd-Steinberg 误差扩散抖动量化模式。处理每个像素后，必须将量化误差按以下比例扩散到相邻像素：
- 右侧 (x+1, y): 7/16
- 左下 (x-1, y+1): 3/16
- 正下 (x, y+1): 5/16
- 右下 (x+1, y+1): 1/16

误差扩散计算必须使用 Float32Array 存储中间结果，每次访问像素值时必须 clamp 到 [0, 255] 范围。最近邻查找时 float 值必须四舍五入到整数再查表。

#### 场景:抖动效果
- **当** 输入图片包含灰色渐变区域
- **那么** 量化结果必须呈现黑白像素交替的抖动图案，视觉上模拟灰度效果

#### 场景:边界不越界
- **当** 处理最右列像素
- **那么** 右侧 (x+1, y) 的误差扩散必须跳过（不越界）
- **当** 处理最底行像素
- **那么** 所有向下方向的误差扩散必须跳过

### 需求:Floyd-Steinberg 阈值保护
量化引擎的 Floyd-Steinberg 抖动模式必须支持**阈值保护**：当像素到最近调色板颜色的 RGB 欧氏距离平方小于阈值时，直接映射到该调色板颜色，**不执行误差扩散**。

阈值必须通过可选的 `ditherThreshold` 参数传入。当未传入时，使用模块级默认值 `DITHER_THRESHOLD_SQ = 24000`。

函数签名：
```typescript
quantizeFloydSteinberg(
  rgba: Uint8Array,
  width: number,
  height: number,
  ditherThreshold?: number   // 可选参数，默认 DITHER_THRESHOLD_SQ
): Uint8Array
```

阈值保护的目的是消除文字区域因微小误差累积产生的彩色杂点，同时保留非文字区域（图片、渐变等）的正常抖动效果。

`DITHER_THRESHOLD_SQ` 默认值为 `24000`（≈ ±89 per channel），该高阈值覆盖 Chromium 渲染中文字体时产生的灰色反锯齿像素（distSq 范围 ~9000-20000）。

#### 场景:接近调色板色的像素跳过抖动
- **当** 输入像素为 (20, 20, 20)（接近纯黑）
- **那么** 到 BLACK(0,0,0) 的 distSq = 1200 < 24000，直接映射为 BLACK(0)，不扩散误差

#### 场景:接近白色的灰边像素跳过抖动
- **当** 输入像素为 (230, 230, 230)（接近纯白，典型文字抗锯齿灰边）
- **那么** 到 WHITE(255,255,255) 的 distSq = 1875 < 24000，直接映射为 WHITE(1)，不扩散误差

#### 场景:中间色像素正常抖动
- **当** 输入像素为 (128, 128, 128)（中灰）
- **那么** 到最近色 WHITE 的 distSq = 48387 > 24000，正常执行 FS 误差扩散

#### 场景:阈值对所有调色板色生效
- **当** 输入像素接近任一调色板颜色（不仅限于黑白）
- **那么** 阈值保护同样生效，彩色文字（红色标题、蓝色链接等）同样受益

#### 场景:未传阈值参数时使用默认值
- **当** 调用 `quantizeFloydSteinberg(rgba, w, h)` 不传 `ditherThreshold`
- **那么** 行为与修改前完全一致，使用 `DITHER_THRESHOLD_SQ = 24000`

#### 场景:传入自定义阈值
- **当** 调用 `quantizeFloydSteinberg(rgba, w, h, 0)`
- **那么** 阈值保护完全禁用，所有像素都执行误差扩散

#### 场景:传入高阈值
- **当** 调用 `quantizeFloydSteinberg(rgba, w, h, 50000)`
- **那么** 大部分像素跳过误差扩散，退化为接近最近邻量化的效果

### 需求:阈值常量导出
量化引擎必须导出 `DITHER_THRESHOLD_SQ` 命名常量，默认值为 `24000`。该常量定义 FS 抖动的阈值保护边界（RGB 欧氏距离平方）。

#### 场景:常量可导入
- **当** 其他模块或测试导入 quantizer.ts
- **那么** 必须能获取 `DITHER_THRESHOLD_SQ` 的值

### 需求:带距离的最近邻查找
量化引擎必须导出 `nearestPaletteIndexWithDist(r, g, b)` 函数，返回 `{ palIdx: number; distSq: number }`，同时提供最近调色板索引和距离平方值。

#### 场景:返回值包含索引和距离
- **当** 调用 `nearestPaletteIndexWithDist(255, 0, 0)`
- **那么** 返回 `{ palIdx: 3, distSq: 0 }`（精确匹配 RED）

#### 场景:与 nearestPaletteIndex 结果一致
- **当** 对任意 RGB 输入调用 `nearestPaletteIndexWithDist`
- **那么** 返回的 `palIdx` 必须与 `nearestPaletteIndex` 对相同输入的返回值一致

### 需求:灰色像素预处理
量化引擎必须在 Floyd-Steinberg 抖动**之前**对灰色像素进行二值化预处理。灰色判定和二值化的阈值必须通过可选参数传入。

函数签名：
```typescript
preprocessGrayPixels(
  rgba: Uint8Array,
  width: number,
  height: number,
  graySpread?: number,          // 可选参数，默认 GRAY_SPREAD_THRESHOLD
  grayLuminanceMidpoint?: number // 可选参数，默认 GRAY_LUMINANCE_MIDPOINT
): Uint8Array
```

当像素的 RGB 通道极差（max - min）≤ `GRAY_SPREAD_THRESHOLD`（默认 40）时，判定为无色相灰色像素，按平均亮度二值化为纯黑 (0,0,0) 或纯白 (255,255,255)（亮度阈值 `GRAY_LUMINANCE_MIDPOINT` = 128）。

此预处理消除 Chromium 渲染中文字体时产生的灰色反锯齿边缘像素，是消除文字杂色的核心手段。

#### 场景:暗灰色二值化为黑色
- **当** 输入像素为 (80, 80, 80)（典型文字边缘灰色）
- **那么** spread = 0 ≤ 40，avg = 80 < 128，二值化为 BLACK (0,0,0)

#### 场景:亮灰色二值化为白色
- **当** 输入像素为 (200, 200, 200)
- **那么** spread = 0 ≤ 40，avg = 200 ≥ 128，二值化为 WHITE (255,255,255)

#### 场景:有色相像素不受影响
- **当** 输入像素为 (255, 0, 0)（纯红）
- **那么** spread = 255 > 40，保持不变

#### 场景:未传参数时使用默认值
- **当** 调用 `preprocessGrayPixels(rgba, w, h)` 不传额外参数
- **那么** 行为与修改前完全一致，使用 `GRAY_SPREAD_THRESHOLD = 40` 和 `GRAY_LUMINANCE_MIDPOINT = 128`

#### 场景:传入自定义灰色极差
- **当** 调用 `preprocessGrayPixels(rgba, w, h, 80, 128)`
- **那么** 通道极差 ≤ 80 的像素被判定为灰色并二值化（比默认 40 更宽松）

#### 场景:传入 0 禁用灰色预处理
- **当** 调用 `preprocessGrayPixels(rgba, w, h, 0, 128)`
- **那么** 只有完全无色差（R=G=B）的像素被二值化，实质上接近禁用预处理

### 需求:阈值保护与灰色预处理单元测试
量化引擎必须包含以下测试用例：

#### 场景:接近黑色的像素无杂色
- **当** 输入 8×8 纯色图片，颜色为 (20, 20, 20)
- **那么** FS 抖动结果全部为 BLACK(0)，无任何其他颜色

#### 场景:接近白色的像素无杂色
- **当** 输入 8×8 纯色图片，颜色为 (240, 240, 240)
- **那么** FS 抖动结果全部为 WHITE(1)，无任何其他颜色

#### 场景:中灰区域仍然抖动
- **当** 输入 8×8 纯色图片，颜色为 (128, 128, 128)
- **那么** FS 抖动结果包含至少两种不同颜色（正常抖动行为未被阈值破坏）

#### 场景:灰色预处理暗灰→黑
- **当** 输入像素 (50, 50, 50)
- **那么** `preprocessGrayPixels` 输出 (0, 0, 0)

#### 场景:灰色预处理亮灰→白
- **当** 输入像素 (200, 200, 200)
- **那么** `preprocessGrayPixels` 输出 (255, 255, 255)

#### 场景:灰色预处理保留彩色
- **当** 输入像素 (255, 0, 0)
- **那么** `preprocessGrayPixels` 输出不变

### 需求:饱和度预处理
量化引擎必须在量化前支持饱和度增强预处理，默认增强系数为 1.4。增强系数必须可配置（1.0 = 不增强）。

饱和度增强采用逐像素 RGB→HSL 转换，将 S（饱和度）分量乘以增强系数后 clamp 到 [0, 1]，再转回 RGB。

#### 场景:默认饱和度增强
- **当** 调用 `enhanceSaturation` 不传系数时
- **那么** 必须以 1.4 倍增强饱和度

#### 场景:系数为1时无变化
- **当** 饱和度增强系数为 1.0
- **那么** 输出像素值必须与输入相同（或在舍入精度内一致）

#### 场景:纯灰色像素不受影响
- **当** 输入像素饱和度为 0（纯灰色/黑色/白色）
- **那么** 增强后颜色不变

### 需求:物理缓冲区编码
量化引擎必须将 480×800 的颜色索引数组编码为 192,000 字节的物理帧缓冲区。编码必须执行 rotation=3 (270°) 坐标变换和 4-bit 打包。

变换公式：逻辑坐标 (lx, ly) → 物理坐标 (px, py) = (ly, 479 - lx)
物理地址：addr = floor(px / 2) + py × 400
打包规则：偶数 px → 高 4 位，奇数 px → 低 4 位

#### 场景:缓冲区大小
- **当** 编码 480×800 的索引数组
- **那么** 输出的 Uint8Array 长度必须为 192,000 字节

#### 场景:坐标变换正确性
- **当** 逻辑坐标 (0, 0) 的颜色索引为 3（RED）
- **那么** 物理坐标 (px=0, py=479)，地址 = 0/2 + 479 × 400 = 191600，高 4 位必须为 3

#### 场景:4-bit打包正确性
- **当** 逻辑坐标 (0, 0) 索引为 3，(0, 1) 索引为 5
- **那么** 物理坐标分别为 (0, 479) 和 (1, 479)
- **那么** 地址 191600 处的字节值必须为 `(3 << 4) | 5 = 0x35`

### 需求:输入格式
量化引擎所有函数接收 RGBA 格式的 Uint8Array 像素数据（每像素 4 字节：R, G, B, A），以及宽度和高度参数。Alpha 通道在量化时忽略。

#### 场景:RGBA输入
- **当** 量化函数接收 Uint8Array 数据
- **那么** 必须按每 4 字节一组读取 R, G, B（跳过 A），行优先顺序

### 需求:单元测试覆盖
量化引擎必须包含以下测试用例：

#### 场景:调色板测试
- 6 色调色板 RGB 值正确
- 索引 4 为 null
- 颜色名称数组正确

#### 场景:最近邻量化测试
- 纯色像素映射到正确索引
- 所有 6 种调色板颜色自身映射正确
- 中间色映射到最近颜色

#### 场景:Floyd-Steinberg 测试
- 纯白图片量化结果全为 WHITE(1)
- 纯黑图片量化结果全为 BLACK(0)
- 灰色区域产生抖动图案（不全是同一颜色）

#### 场景:缓冲区编码测试
- 输出长度 192,000
- 单像素坐标变换正确
- 全白/全黑缓冲区填充值正确（全白 = 0x11，全黑 = 0x00）
