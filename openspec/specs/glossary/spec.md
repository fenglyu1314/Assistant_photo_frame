# 术语表 (Glossary)

> 定义 Assistant Photo Frame 项目中所有核心术语的标准用法。
> 所有开发文档、OpenSpec artifact、代码注释和日常沟通必须遵循本术语表。

---

## 核心术语

### 墨水屏

| 属性 | 值 |
|------|-----|
| **英文** | e-Paper / e-Paper Device |
| **指代** | 硬件整机——包含 ESP32-S3 主板、7.3" 墨水屏面板、相框外壳的完整设备 |
| **使用场景** | "推送到墨水屏"、"墨水屏已连接"、"墨水屏显示异常" |
| **注意** | 在本项目中，「墨水屏」默认指整机而非单独的屏幕面板 |

**示例**:
- ✅ "桌面端将画面推送到墨水屏"
- ✅ "墨水屏通过 USB 连接到电脑"
- ❌ "把数据发到 ESP32 上" （应说"推送到墨水屏"）
- ❌ "发到设备上" （太笼统，应说"推送到墨水屏"）

---

### 桌面端

| 属性 | 值 |
|------|-----|
| **英文** | Desktop / Desktop App |
| **指代** | 运行在 PC 上的 Electron 应用程序，负责渲染、量化、内容管理和推送 |
| **代码目录** | `companion/` |
| **使用场景** | "桌面端渲染画面"、"在桌面端编辑待办"、"桌面端发送帧数据" |
| **正式全称** | 墨水屏桌面端（仅在需要完整上下文时使用） |

**示例**:
- ✅ "桌面端负责渲染和量化"
- ✅ "在桌面端添加一个新功能"
- ❌ "Companion 应用"（内部目录名，不用于沟通）
- ❌ "PC 端"（可用但不推荐，优先使用"桌面端"）

---

### 固件

| 属性 | 值 |
|------|-----|
| **英文** | Firmware |
| **指代** | 运行在 ESP32-S3 上的程序，负责接收帧数据、写入 PSRAM、驱动墨水屏刷屏 |
| **代码目录** | `firmware/` |
| **使用场景** | "固件接收帧数据"、"烧录固件"、"固件版本" |

---

### 帧协议

| 属性 | 值 |
|------|-----|
| **英文** | Frame Protocol |
| **指代** | 桌面端与墨水屏之间的 USB CDC 二进制通信协议（MAGIC 0xEB0D） |
| **使用场景** | "帧协议定义了数据格式"、"通过帧协议发送分块数据" |
| **注意** | 不要简称为"协议"以免与其他协议（如 HTTP）混淆 |

---

## 子组件术语

以下术语仅在讨论硬件细节或特定模块时使用：

### EPD 面板

| 属性 | 值 |
|------|-----|
| **英文** | EPD Panel |
| **指代** | 墨水屏中的 7.3" 电子纸显示面板（不含主板和外壳） |
| **使用场景** | 仅在讨论屏幕硬件特性时使用，如"EPD 面板的刷新率"、"EPD 面板的色域" |
| **注意** | 日常开发中说"墨水屏"即可，只有讨论面板本身的硬件特性时才需要用这个词 |

### 相框外壳

| 属性 | 值 |
|------|-----|
| **英文** | Frame Shell |
| **指代** | 墨水屏的物理外壳/支架部分 |
| **使用场景** | 仅在讨论外观、尺寸、散热等物理结构时使用 |

### ESP32 主板

| 属性 | 值 |
|------|-----|
| **英文** | ESP32 Board |
| **指代** | 墨水屏中的 ESP32-S3 开发板 |
| **使用场景** | 仅在讨论芯片规格、GPIO、内存等硬件细节时使用 |
| **注意** | 不要用"ESP32"代指整机，整机请用"墨水屏" |

## 系统级术语

### 墨水屏桌面助手

| 属性 | 值 |
|------|-----|
| **英文** | Assistant Photo Frame |
| **指代** | 本项目/产品的整体，包含墨水屏 + 桌面端 + 固件的完整系统 |
| **使用场景** | 对外介绍、README、产品描述 |

### 帧缓冲区

| 属性 | 值 |
|------|-----|
| **英文** | Frame Buffer |
| **指代** | 192,000 字节的显示数据（800/2 × 480，4-bit packed），存储在 PSRAM 中 |
| **使用场景** | "帧缓冲区写入 PSRAM"、"编码帧缓冲区"、"分块传输帧缓冲区" |

### 渲染管线

| 属性 | 值 |
|------|-----|
| **英文** | Render Pipeline |
| **指代** | 桌面端中从数据获取到帧数据生成的完整流程：数据 → HTML渲染 → 截屏 → 量化 → 编码 |
| **使用场景** | "渲染管线执行一次刷新"、"渲染管线的量化阶段" |

### 量化

| 属性 | 值 |
|------|-----|
| **英文** | Quantization |
| **指代** | 将全彩图像转换为 6 色（BLACK/WHITE/YELLOW/RED/BLUE/GREEN）的过程 |
| **使用场景** | "量化引擎"、"Floyd-Steinberg 抖动量化" |

## 术语对照速查

| 你可能会说的 | 应该说 | 原因 |
|-------------|--------|------|
| ESP32、设备、开发板 | **墨水屏** | "墨水屏"指整机 |
| Companion、PC端、桌面应用 | **桌面端** | 统一称呼 |
| 协议 | **帧协议** | 避免与其他协议混淆 |
| 屏幕 | **墨水屏**（整机）或 **EPD 面板**（仅面板） | 根据语境选择 |
| 相框 | **墨水屏**（整机）或 **相框外壳**（仅外壳） | 根据语境选择 |
